# Linux Practice – Service Inspection (Nginx)

## 🔎 Process Checks

### 1. Check running processes

```bash
ps -ef | grep nginx
```

**Purpose:** Verifies that nginx master and worker processes are running.

---

### 2. Find process using PID lookup

```bash
pgrep -a nginx
```

**Purpose:** Displays process IDs and command details for nginx.

---

## ⚙️ Service Checks

### 3. Check service status

```bash
sudo systemctl status nginx
```

**Purpose:** Confirms whether the service is active, shows uptime, PID, and recent logs.

---

### 4. Check if service is enabled at boot

```bash
sudo systemctl is-enabled nginx
```

**Purpose:** Verifies whether nginx starts automatically when the system boots.

---

### 5. List all running services

```bash
sudo systemctl list-units --type=service --state=running
```

**Purpose:** Displays all currently active services.

---

## 📜 Log Checks

### 6. Check systemd journal logs

```bash
sudo journalctl -u nginx --no-pager | tail -20
```

**Purpose:** Shows the last 20 log entries related to nginx.

---

### 7. Check nginx access logs

```bash
sudo tail -n 20 /var/log/nginx/access.log
```

**Purpose:** Displays recent incoming HTTP requests.

---

### 8. Check nginx error logs

```bash
sudo tail -n 20 /var/log/nginx/error.log
```

**Purpose:** Displays recent error messages generated by nginx.

---

## 🛠 Mini Troubleshooting Steps

### Scenario: Website not loading

1. **Check service status**

   ```bash
   sudo systemctl status nginx
   ```

   If inactive → Start it:

   ```bash
   sudo systemctl start nginx
   ```

2. **Check if port 80 is listening**

   ```bash
   sudo ss -tulnp | grep :80
   ```

3. **Check error logs**

   ```bash
   sudo tail -n 20 /var/log/nginx/error.log
   ```

4. **Restart service after configuration changes**

   ```bash
   sudo systemctl restart nginx
   ```

5. **Verify in browser**

   ```
   http://65.0.89.127/
   ```
